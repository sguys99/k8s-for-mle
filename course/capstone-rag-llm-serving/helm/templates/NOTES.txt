§1. Pod Ready 대기 (~6~8 분 — vLLM cold start 가 가장 길음)

  kubectl get pods -n {{ .Values.namespace }} -w
  # vLLM startupProbe 60×10s = 최대 10 분. 첫 install 시 모델 5GB 다운로드 포함.
  # 두 번째 기동(PVC 캐시 hit) 은 30 초 안에 ready.

§2. Service / Endpoint 확인

  kubectl get svc -n {{ .Values.namespace }}
  # qdrant (Headless 6333/6334), vllm (ClusterIP 8000), rag-api (ClusterIP 8001)

{{- if .Values.ingress.enabled }}

§3. Ingress IP 확인 + host 갱신 (LoadBalancer 프로비저닝 ~3~5 분)

  EXTERNAL_IP=$(kubectl get ing rag-api -n {{ .Values.namespace }} -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
  echo "External IP: $EXTERNAL_IP"
  helm upgrade {{ .Release.Name }} . -n {{ .Values.namespace }} \
    -f values-prod.yaml --set ingress.host=$EXTERNAL_IP.nip.io

§4. RAG API end-to-end 검증 (1 줄 완료 기준)

  curl http://$EXTERNAL_IP.nip.io/chat \
    -H 'Content-Type: application/json' \
    -d '{"messages":[{"role":"user","content":"K8s에서 GPU 어떻게 잡지?"}],"top_k":3}' | jq

  # 기대: 200 OK + answer 텍스트 + sources 3 개 (인용 마커 [1], [2], [3] 포함)
{{- else }}

§3. RAG API end-to-end 검증 (port-forward, 1 줄 완료 기준)

  kubectl port-forward -n {{ .Values.namespace }} svc/rag-api 8001:8001 &
  curl http://localhost:8001/chat \
    -H 'Content-Type: application/json' \
    -d '{"messages":[{"role":"user","content":"K8s에서 GPU 어떻게 잡지?"}],"top_k":3}' | jq

§5. prometheus-adapter 별도 설치 (HPA 커스텀 메트릭)

  본 차트는 adapter values 만 ConfigMap 으로 동봉합니다. 학습자가 별도 helm install:

  kubectl get cm prometheus-adapter-values -n {{ .Values.namespace }} \
    -o jsonpath='{.data.values\.yaml}' > /tmp/adapter-values.yaml
  helm install prometheus-adapter prometheus-community/prometheus-adapter \
    -n monitoring -f /tmp/adapter-values.yaml --version 4.10.0 --wait

  # 검증: kubectl get --raw "/apis/custom.metrics.k8s.io/v1beta1" | jq | grep vllm_num_requests_running
{{- end }}
